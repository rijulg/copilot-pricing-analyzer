Token profile Estimated tokens per prompt for an agentic coding turn (like the prompts that built this app). Shared across all models. Per 1M tokens.

Defaults reflect a typical agent turn: a short new instruction (~9K), a large slice of cached context — system prompt, tool definitions, conversation history and re-read source files (~34K) — fresh content written to the cache (~11K), and a few thousand output tokens of generated code and prose. Context growth is the key one: every turn the prior output, tool results and new file reads (~12K) pile onto the context, so each prompt re-reads more than the last.

The maths How every number on this page is computed.

1 · Cost of the first prompt

Each model has four per-token prices (quoted per 1M tokens): input pin, cached input pcache, cache write pwrite and output pout. With the shared token profile — tin, tcache, twrite, tout tokens — the first prompt (before any history piles up) costs:

base = ( tin·pin + tcache·pcache + twrite·pwrite + tout·pout ) / 1,000,000

The division by 1,000,000 converts the per-1M prices into a per-token rate.

2 · Context explosion & total cost

An agent doesn't start fresh each turn — the whole conversation is resent. Every prompt adds about g tokens of history (the previous output, tool results, freshly read files), which the next prompt re-reads as cached input. So the k-th prompt costs the base plus the surcharge for the k−1 turns of accumulated context:

cost(k) = base + (k−1)·δ   where   δ = g·pcache / 1,000,000

Summing that arithmetic series over n prompts gives a quadratic total — the “explosion”. Each extra prompt is pricier than the one before, so costs bend upward rather than growing in a straight line:

costtotal = n·base + δ · n(n−1)/2

This is the value the ranking sorts on — the cheapest total wins. A model with a higher base price can still win if it finishes in fewer prompts, because it dodges most of the context explosion.

3 · Break-even point

Take the model with the highest base cost as the reference (E) and its chosen total costtotalE. Because a cheaper model C's cost is now quadratic, break-even is the positive root of:

(δC/2)·n² + (baseC − δC/2)·n − costtotalE = 0

Read it as: if the cheaper model C needs more than n prompts to do what E does in its chosen number of prompts, the pricier model E is the better deal. On the chart this is where each cheaper model's upward-bending cost curve crosses the dashed reference line of the most expensive model. (With no context growth, δ=0 and it collapses back to the simple costtotalE / baseC.)
