Token profile Estimated tokens per prompt for an agentic coding turn (like the prompts that built this app). Shared across all models. Per 1M tokens.

Defaults reflect a typical agent turn: a short new instruction (~9K), a large slice of cached context — system prompt, tool definitions, conversation history and re-read source files (~34K) — fresh content written to the cache (~11K), and a few thousand output tokens of generated code and prose.

The maths How every number on this page is computed.

1 · Cost of a single prompt

Each model has four per-token prices (quoted per 1M tokens): input pin, cached input pcache, cache write pwrite and output pout. With the shared token profile — tin, tcache, twrite, tout tokens — the cost of one prompt is:

costprompt = ( tin·pin + tcache·pcache + twrite·pwrite + tout·pout ) / 1,000,000

The division by 1,000,000 converts the per-1M prices into a per-token rate.

2 · Total cost of a task

If a model needs n prompts to finish the task, its total cost is simply the per-prompt cost repeated:

costtotal = n · costprompt

This is the value the ranking sorts on — the cheapest total wins, even if its per-prompt price is higher, because it may need fewer prompts.

3 · Break-even point

Take the model with the highest per-prompt cost as the reference (call it E) and its chosen total costtotalE. A cheaper-per-prompt model C reaches that same spend after:

nbreak-even = costtotalE / costpromptC

Read it as: if the cheaper model C needs more than nbreak-even prompts to do what E does in its chosen number of prompts, the pricier model E is the better deal. On the chart this is where each cheaper model's cost line crosses the dashed reference line of the most expensive model.
